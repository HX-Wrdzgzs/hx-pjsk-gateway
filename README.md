# hx-pjsk-gateway

hx-pjsk-gateway 是 Project Sekai 网页查询终端与数据可视化网关，当前由个人与开发组内部维护。

本项目用于把底层机器人协议节点与网页端查询体验连接起来，让用户通过 Web 页面完成 PJSK 数据查询、身份确认和结果查看。

- 使用范围：面向现有机器人、网关和前端环境，不保证外部环境可直接运行。

## 项目定位

`hx-pjsk-gateway` 是一个 Web Gateway，而不是单纯的机器人插件。它位于用户浏览器、身份确认节点、数据节点之间，负责请求转发、异步响应匹配和结果格式化。

```text
Browser / Web UI
        ↓
Gateway API
        ↓
Bot-side verification
        ↓
Data bridge
        ↓
Haruki / Sakura data node
```

## 技术栈与架构

- 前端：React + Vite + TypeScript + Zustand
- 网关层：FastAPI + asyncio / uvloop
- 数据节点：Haruki / Sakura Client / OneBot v11
- 鉴权节点：NoneBot2
- 存储与状态：MySQL + Redis
- 边缘与防护：Cloudflare Pages、Cloudflare Tunnel、Turnstile、WAF、AuthLit4

## 核心机制

### 强身份确认

项目不采用传统账号密码体系，而是通过机器人侧验证流程确认用户身份，并把网页会话与机器人侧已有数据绑定。

### 异步查询与响应匹配

Gateway 使用请求标识隔离并发查询，避免多个网页请求之间发生响应串包。

### 图片旁路处理

机器人侧返回本地图片路径时，Gateway 可以进行格式转换和临时资源整理，使网页端可以稳定展示结果。

## 安全边界

- Web 查询业务与机器人连接链路解耦，降低高频 Web 请求直接冲击机器人侧的风险。
- 验证码应具备有效期，并绑定会话上下文。
- JWT / Session Token 不应在日志中明文输出。
- 内部数据节点不应直接暴露到公网。
- Cloudflare Tunnel 只负责入口隐藏，不应替代应用层鉴权。
- 图片旁路目录应限制访问范围，避免任意文件读取。

## 阶段性路线

### Phase 1：核心协议层

- 搭建 Gateway 基础框架。
- 建立与数据节点的守护连接。
- 验证异步请求挂起与唤醒机制。

### Phase 2：并发控制与资源层

- 引入软超时与降级返回。
- 实装任务状态查询。
- 接入图片路径捕获、转码与静态资源挂载。

### Phase 3：认证与状态中心

- 部署机器人侧验证插件。
- 跑通验证码状态机。
- 完成状态推送。

### Phase 4：业务全量接入

- 完成 React 前端工程化。
- 接入个人中心、查卡图鉴、榜线仪表盘、组卡计算等视图。
- 完善异常提示、重试与任务恢复。

## 维护说明

新增功能时应优先保证：

- 身份映射正确。
- 并发请求不会串包。
- 失败请求可恢复或明确报错。
- 敏感配置不进入前端包。
- 机器人侧返回内容经过必要清洗后再展示。
